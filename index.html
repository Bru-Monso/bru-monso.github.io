Caso real

Gobierno de identidades con Microsoft Entra ID

Contexto

Microsoft Entra ID ya estaba implantado, pero todavía había margen para mejorar la gestión de identidades, permisos y acceso a las aplicaciones corporativas.

El objetivo no era implantar una única funcionalidad, sino construir una base más sólida sobre la que seguir creciendo.

Retos

  • Integrar aplicaciones mediante Single Sign-On.
  • Sincronizar información del Active Directory local hacia Entra ID.
  • Automatizar la asignación de licencias mediante grupos dinámicos.
  • Mejorar la autenticación mediante políticas de Conditional Access.
  • Gestionar colaboradores externos mediante usuarios B2B.
  • Preparar el entorno para futuras implantaciones como Privileged Identity Management.

Casos destacados

Próximos pasos

Implementación de PIM y mejora continua del modelo de gobierno de identidades.

Capítulo 01

Implantación de Single Sign-On

Contexto

Algunas aplicaciones empresariales necesitaban integrarse con Microsoft Entra ID para centralizar el acceso y reducir la gestión manual de credenciales.

Qué había que resolver

Revisar la aplicación, entender qué requería el proveedor, configurar SAML, validar atributos, probar el acceso y pasarlo a producción sin romper el uso diario.

Resultado

Aplicaciones integradas con SSO, acceso más ordenado y una base más clara para futuras integraciones.

Capítulo 02

Sincronización de atributos desde Active Directory

Contexto

Había información académica y organizativa que podía ser útil en Entra ID, pero primero debía estar bien informada y controlada desde el entorno local.

Qué había que resolver

Informar atributos en Active Directory, controlar la lógica de actualización y permitir que esa información llegara a Entra ID para usarla en grupos, licencias y organización.

Resultado

Una clasificación más útil para automatizar decisiones posteriores sin depender tanto de procesos manuales.
